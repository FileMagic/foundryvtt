Overview

This guide covers common troubleshooting issues as well as general troubleshooting steps.

It's important to first determine what the possible cause of the issue is. Not all steps will apply to each case. You should update your Game Systems and Add-on Modules before starting this process, then check if that fixed the issue you are facing.

Startup and Setup Screen Issues

Several different factors can cause Foundry VTT to not start. @Article[ts-doesnt-start] covers solutions to these in detail, including Foundry VTT not starting at all or failing to fully load.

Reset World Passwords

Resetting the passwords to a World requires only a few steps. They are detailed in @Article[reset-user-passwords].

My Modules or Game Systems Won't Update

There are a few reasons that would prevent Add-on Modules or Game Systems from updating:

  • Host Issues: Modules and Game Systems are hosted by various providers. If a provider is facing technical issues or heavy traffic, it may refuse your connection. Try waiting a few minutes and trying again. If you're using a VPN, try disabling it and trying again.
  • Bad Manifest: This is an issue with one of the module or system's files. We suggest reaching out to the author of the Module or Game System for assistance.
  • No Update Available for Your Version of Foundry VTT: If a module or system author has only created updates for a newer major version of Foundry VTT, then it will fail to update. The module or system will still work the same as before. If you wish to upgrade Foundry VTT and use the latest module release, first back up your User Data following the steps at @Article[user-data-backup].
  • Permissions Issue: If an error starting with "EPERM" appeared when you tried to update, you will need to change the permissions level of your User Data folder. This process depends on your operating system. A step-by-step guide is available in @Article[ts-common-errors].
My World Won't Launch

Worlds not launching typically come down to database or permissions errors. A red error banner should appear at the top of the screen indicating what the specific error is. @Article[ts-common-errors] explains these errors and their solutions in more detail.

Connection Issues

Foundry VTT needs a clear path to accept incoming connections. A closed connection or players being unable to connect is usually due to something preventing traffic from reaching Foundry VTT on your computer. Once you've determined the possible source, the article @Article[ts-connection-closed] contains further instructions on identifying and resolving network related issues.

Determine Source of Connection Issues

Local Connectivity Test

Using another device on the same network as your computer, such as a laptop or phone on WiFi, attempt to connect to Foundry VTT using the Local Network invitation link. If Foundry VTT does not load, something on your computer is preventing incoming connections. Typically this happens if you have a firewall active without an exception for Foundry VTT, antivirus software, or network settings on Windows. See @Article[port-forwarding] for more information on creating a firewall rule. If you are using antivirus software, temporarily disable it and try connecting again. If you're able to connect, refer to its documentation for steps to add an exception for Foundry VTT.

Local Connections Work

If Foundry VTT loads with the local connectivity test but reports your connection is closed, or your players are still unable to connect, then the issue will either be related to port forwarding or ISP technology known as CGNAT. See @Article[port-forwarding] for instructions on how to set up port forwarding and check that existing rules are working properly. @Article[ts-connection-closed] contains further instructions on troubleshooting a closed connection, including testing for CGNAT.

I'm Using a VPN

VPNs add extra layers of routers and servers between you and your players. Unfortunately they also act as black boxes that are inaccessible for troubleshooting. We advise disabling your VPN when hosting Foundry Virtual Tabletop.

General Troubleshooting Steps

Troubleshooting other issues is a fairly straightforward process. These include disabling all modules, checking the console for errors, and narrowing down possible causes for the issue.

Disable all modules by launching your world in safe configuration. @Article[launch-safe-configuration] contains the steps for this process. If this fixes the problem, the issue was module related. Re-enable your modules in small batches, checking each time if the issue persists, to quickly determine the module at fault. The developer console may indicate which module is at fault too.

My Issue Is Still There After Disabling Modules

The next step is to check the developer console for error messages. This involves opening your browser's Developer Tools, navigating to its Console tab, and then looking for messages in red text. Warnings in yellow can usually be ignored, most issues will show as errors. Checking for errors will help guide troubleshooting and identify the culprit. @Article[ts-common-errors] contains a list of the most common errors you may see, as well as steps to resolve them.

  • Chrome/Edge: Press 'F12' or 'Ctrl + Shift + J' on Windows, or 'Opt + Cmd + J' on MacOS .
  • Firefox: Press 'F12' or 'Ctrl + Shift + I' on Windows, or 'Opt + Cmd + I' on MacOS.
Warnings vs Errors

Foundry VTT may show both warnings and errors. Warnings indicate that something is being done incorrectly. Most warnings thus are specific to Game System and Add-on Module developers and can be safely ignored. Errors however indicate that something has failed to work properly. This article covers several warnings that are relevant to end users, but otherwise focuses on errors.

Rule Out Browser Related Issues

I Am Self-Hosting
  1. Launch Foundry VTT as normal.
  2. Open an incognito or private tab in a web browser and go to http://localhost:30000 to connect to Foundry VTT.
  3. The Issue Does Not Persist: The cause of the issue is likely a problem with browser extensions or the standalone Foundry VTT application. You can use Foundry VTT normally by accessing it via web browser.
  4. The Issue Persists: The issue is likely not browser related. Proceed to the next section.
I Use a Remote Host
  • Connect to Foundry VTT as normal using either a private browser window, or a browser you don't commonly use.
  • The Issue Does Not Persist: A browser extension may be interfering with Foundry VTT, or you're using an unsupported browser. Foundry VTT supports Chrome, Edge, Opera (not Opera GX), and Firefox.
  • The Issue Persists: The issue is likely not browser related. Proceed to the next section.

    • Rule Out World, System, or Foundry VTT Specific Issues

    If the issue persists after disabling all modules, the next step will be determining if it's an issue with the World itself, the Game System, or with Foundry VTT. Update After each step, check for errors in the developer console. Do not enable any modules during this process.

    1. Return to Setup and create a new World with the same game system.
    2. Launch this World and attempt to reproduce the issue.
    3. The Issue Does Not Persist: The cause of the issue is related to something in the World itself.
    4. The issue Persists: Create a new World in a different Game System. Attempt to reproduce the issue again. If it persists, the issue is likely a problem with Foundry VTT itself. If not, the issue is with the system you're using in the affected World.
    Resolving World, Game System, or Foundry VTT Issues
    • World Issues: Check the console for errors. If any exist, check if they're listed in the @Article[ts-common-errors] article.
    • Game System Issues: Update the relevant Game System on the Game Systems page, or click "Update All." Check again if the error persists. If it does, check the console for errors and refer to @Article[ts-common-errors].
    • Foundry VTT Specific Issues: Navigate to the Updates page and note what version of Foundry VTT is listed under 'Current Version.' Backup your User Data following the steps from @Article[user-data-backup]. Uninstall Foundry VTT, then download and install either the same version you had installed, or the latest stable version for your major version of Foundry VTT, from your Purchased Licenses page.
