Overview

This article covers troubleshooting steps for common issues in Foundry. After completing each step, check your world to see if the issue remains. If the issue is still there after you've completed all the Basic Troubleshooting steps, we strongly recommend that you contact us or join our Discord community for live support.

Troubleshooting Steps

Basic Troubleshooting

  1. Refresh and Clear Browser Cache: With the world loaded, hold down 'Ctrl'/'Cmd'and press 'F5' to refresh Foundry.
  2. Launch World in Safe Configuration: While modules are a popular feature of Foundry, conflicts between them or outdated modules can lead to unexpected problems. Launching your world in Safe Configuration is the fastest and easiest way to check if modules are the cause of your problem. To do this, navigate to the Worlds screen and find the world with issues. Press 'Edit World' and then check 'Launch in Safe Configuration'. When you launch the world, Foundry will disable all modules, deactivate all scenes, and stop playing all tracks. To save time, please do not re-enable any modules until you are finished troubleshooting.
  3. Restart Foundry: If you're using the standalone Foundry application, close out of Foundry entirely. Then launch Foundry again. Load the world and check if the issue remains. If you're using a Hosting Partner, follow their steps to restart Foundry.
  4. Update Systems and Modules: Return to Setup and navigate to the Game Systems screen. Press the 'Update All' button to update your game systems to their latest version.
  5. Use an Incognito Window or Different Browser: Sometimes unsupported browsers or certain browser extensions interfere with Foundry. Connect to your Foundry instance in an incognito/private tab, or a different browser. If you're using the standalone Foundry application, go to http://localhost:30000 in your web browser to directly connect to Foundry.
  6. Reinstall Game System: Return to Setup and navigate to the Game Systems screen. Find the game system your world uses, eg 'DnD5e', then click on 'Uninstall.' Click 'Yes' when the warning prompt comes up. After the game game system is uninstalled, click 'Install System', navigate to the relevant system, and click 'Install.' Then load the world and check if the issue continues.
  7. Update Foundry: Finally, if the issue still isn't resolved, check that Foundry is up to date with the latest Stable build. Navigate to the Administration screen and check that you're using the Stable release channel. Do not upgrade to a Prototype, Development, or Testing version! Press "Check for Updates", and then "Download" if an update exists.

Advanced Troubleshooting

If the above steps do not resolve the issue then we strongly recommend you contact us or join our Discord community for live support. We may ask that you perform a few more troubleshooting steps to help find the cause of the issue. They are described below for your reference.

  • Check Another Scene: Check that all your modules are disabled and then create a new scene. Check if the issue continues in this scene, or if it's isolated to specific scene(s).
  • Check Another World: Create a new world with the same game system and do not activate any modules. Check in that world if the issue continues. This helps to rule out possible issues with your game world.
  • Try A Different Game System: Click on 'Return to Setup' if your world is active. Create a new world with a different game system. Check if the issue continues to happen in this world. Similar to checking another world and scene, this helps rule out an issue with your game system.
  • Check the Console: Press F12 to open up the Developer Tools panel. Select the tab titled 'Console' and check for any error messages in red text. If there are any, get a screenshot or copy and paste them into a text file.
  • Check for Errors: You may be asked to provide or check Foundry's error log. To find the error log, navigate to your User Data folder, then open the Logs directory. The error log is named 'error.txt'.
